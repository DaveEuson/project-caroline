# Project Caroline v0.3.0-beta.1

First public beta release.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/Project-Caroline/project-caroline/v0.3.0-beta.1/install.sh | bash -s -- --channel v0.3.0-beta.1
```

## Tested Targets

- Raspberry Pi OS Desktop 64-bit, kiosk mode
- Ubuntu Server 64-bit, server/client mode
- Ubuntu Desktop 64-bit, desktop testing mode

## Highlights

- Fullscreen Caroline kiosk UI with cyberpunk dashboard themes
- Local Ollama mode plus OpenRouter cloud AI mode
- Discord relay for kiosk messages and remote conversation
- Google Calendar add/read/delete support
- Local tasks, feedback capture, memory, and system widgets
- Philips Hue discovery/manual bridge setup and group control
- Spotify login flow with redirect guidance
- Optional local browser login during install
- Clean uninstall/reinstall workflow for beta QA

## Release Notes

- Hardened Discord inbound/outbound handling and settings validation.
- Fixed Hue bridge discovery/manual setup behavior after clean install.
- Preserved quoted Google Calendar event titles and honored requested durations.
- Improved local `gemma3:1b` behavior with latest-message-only context, shorter replies, explicit task-action guards, and fast local responses for common beta/support prompts.
- Added README and promo screenshots for the public beta page.

## Known Limitations

- Local Ollama on small hardware is private and free, but slower and less polished than OpenRouter.
- Caroline is intended for trusted local networks only; do not expose ports directly to the public internet.
- HTTPS uses a local self-signed certificate for microphone/browser security.
- Local browser login is optional; enable it for shared networks.
- The updater follows the installed channel. Tagged installs stay on the tag unless you reinstall from `release` or `nightly`.
